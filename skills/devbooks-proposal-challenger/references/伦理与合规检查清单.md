# 伦理与合规检查清单

> 适用场景：项目涉及个人数据、AI 决策、社会影响时，使用本清单进行伦理审查。
>
> 本文档为**可选参考**，仅当项目涉及以下领域时使用：
> - 个人数据收集/处理/存储
> - 算法决策（推荐、审核、评分）
> - 用户行为追踪
> - 自动化决策影响用户权益

---

## 1) 数据隐私（GDPR/CCPA 合规）

### 1.1 数据收集检查

- [ ] **必要性原则**：只收集实现功能所必需的数据
- [ ] **知情同意**：用户在收集前明确知道并同意
- [ ] **目的限制**：数据仅用于声明的目的，不做二次利用
- [ ] **最小化原则**：收集最少必要的数据字段

### 1.2 数据存储检查

- [ ] **留存期限**：明确数据保留多久，过期后如何删除
- [ ] **加密存储**：敏感数据是否加密存储
- [ ] **访问控制**：谁可以访问这些数据，是否有审计日志
- [ ] **跨境传输**：数据是否传输到其他国家/地区

### 1.3 用户权利检查

- [ ] **查阅权**：用户能否查看自己的数据
- [ ] **更正权**：用户能否更正错误数据
- [ ] **删除权**：用户能否要求删除数据（"被遗忘权"）
- [ ] **可携带权**：用户能否导出自己的数据

### 1.4 design.md 必须声明

```markdown
### 数据隐私声明
- 收集的数据类型：<列出字段>
- 数据用途：<说明目的>
- 留存期限：<xx 天/月/年>
- 用户权利实现：<查阅/更正/删除的入口>
```

## 2) 算法偏见（AI/ML 系统）

### 2.1 偏见风险检查

| 偏见类型 | 表现 | 检测方法 |
|----------|------|----------|
| 历史偏见 | 训练数据反映历史不平等 | 数据审计 |
| 代表性偏见 | 某些群体在数据中代表性不足 | 群体分布统计 |
| 测量偏见 | 特征选择对某些群体不公平 | 公平性指标对比 |
| 评估偏见 | 评估指标对某些群体不公平 | 分群评估 |

### 2.2 公平性检查清单

- [ ] 是否分析了不同人群的模型表现差异？
- [ ] 是否有机制检测和修正偏见？
- [ ] 是否定义了公平性指标（如：各群体准确率差异 < 5%）？
- [ ] 是否有人工复核机制处理边界案例？

### 2.3 可解释性检查清单

- [ ] 用户能否理解"为什么我得到这个结果"？
- [ ] 是否提供了申诉/反馈渠道？
- [ ] 重要决策是否有人工介入机会？

## 3) 社会影响评估

### 3.1 负面影响检查

- [ ] 这个功能是否可能被滥用？如何防止？
- [ ] 是否会造成信息茧房/极化？
- [ ] 是否会加剧数字鸿沟？
- [ ] 是否会影响用户心理健康？

### 3.2 弱势群体保护

- [ ] 未成年人是否有特殊保护措施？
- [ ] 老年人/残障人士是否能无障碍使用？
- [ ] 低收入群体是否被排除在服务之外？

### 3.3 design.md 必须声明

```markdown
### 社会影响评估
- 潜在负面影响：<说明风险>
- 缓解措施：<说明如何降低风险>
- 弱势群体保护：<说明特殊措施>
```

---

## 4) 安全与滥用防护

### 4.1 滥用场景检查

- [ ] 是否考虑了恶意用户的滥用场景？
- [ ] 是否有速率限制防止刷量/攻击？
- [ ] 是否有内容审核机制（如：UGC 内容）？
- [ ] 是否有异常检测机制？

### 4.2 数据泄露防护

- [ ] 敏感数据是否有脱敏处理？
- [ ] API 是否有认证和授权？
- [ ] 日志是否会泄露敏感信息？

## 5) Proposal 阶段伦理检查

在 `devbooks-proposal-challenger` 质疑时，增加以下检查点：

### 5.1 强制检查（涉及个人数据时）

- [ ] **数据必要性**：是否只收集了实现功能必需的数据？
- [ ] **用户知情**：用户是否明确知道数据如何被使用？
- [ ] **退出机制**：用户能否选择不参与或删除数据？

### 5.2 可选检查（涉及算法决策时）

- [ ] **可解释性**：用户能否理解决策原因？
- [ ] **申诉机制**：用户能否对决策提出异议？
- [ ] **公平性**：是否对不同群体公平？

### 5.3 Proposal 模板补充

```markdown
### 伦理与合规（可选，但涉及个人数据时必填）

#### 数据隐私
- 收集的数据：<无 / 列出字段>
- 用户同意机制：<无 / 说明方式>
- 删除/导出能力：<无 / 说明入口>

#### 算法决策（如适用）
- 决策类型：<无 / 推荐 / 审核 / 评分 / 其他>
- 可解释性：<无 / 说明方式>
- 申诉机制：<无 / 说明入口>

#### 社会影响
- 潜在负面影响：<无 / 说明风险>
- 缓解措施：<无 / 说明方式>
```

## 6) 合规法规速查

| 法规 | 适用地区 | 核心要求 |
|------|----------|----------|
| GDPR | 欧盟 | 知情同意、数据最小化、被遗忘权、数据可携带 |
| CCPA | 加州 | 知情权、删除权、不歧视权、opt-out 权 |
| PIPL | 中国 | 单独同意、本地存储、跨境评估 |
| COPPA | 美国 | 13 岁以下儿童数据保护 |
