## 使用指南

### 1. 谁来写 Project Profile？

- **初始化**：brownfield-bootstrap skill 在项目初始化时创建
- **更新**：proposal-author skill 在每次提案时更新
- **维护**：archiver skill 在归档时同步更新

### 2. 谁来读 Project Profile？

| Skill | 读取目的 |
|-------|----------|
| proposal-author | 了解用户画像，推荐合适的方案 |
| design-doc | 了解技术偏好，选择合适的架构 |
| implementation-plan | 了解时间约束，制定合理的计划 |
| archiver | 更新画像，记录决策演变 |

### 3. 画像的作用

- **个性化推荐**：基于用户画像推荐合适的方案
- **决策一致性**：确保所有决策符合用户的长期目标
- **知识沉淀**：记录用户的决策偏好，避免重复询问

### 4. 画像的更新频率

- **初始化**：项目创建时
- **角色升级**：用户需求发生重大变化时
- **新增需求**：发现新的隐藏需求时
- **定期回顾**：每季度回顾一次，确保画像准确
